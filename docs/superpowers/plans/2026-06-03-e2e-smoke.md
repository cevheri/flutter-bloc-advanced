# End-to-End Smoke Test Implementation Plan (#152)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add one headless `integration_test/` e2e smoke (cold start → login → dashboard, mock mode) and run it in CI — putting the declared `integration_test` dependency to real use. Closes #152.

**Architecture:** `integration_test/app_smoke_test.dart` boots the real `App` (default `AppDependencies` = `AppConfig.dev()` = mock mode, serving `assets/mock/*.json`), drives the login form with mock credentials, and asserts the dashboard shell renders. It reuses `TestEnv.reset()` (relative import) for the secure-storage mock since the global `flutter_test_config` does not apply outside `test/`.

**Tech Stack:** Flutter 3.44.0, `integration_test` (SDK), `flutter_test`, FVM. Run with `fvm flutter test integration_test/`.

---

## Key facts

- `const AppDependencies({appConfig = const AppConfig.dev()})` — **default is dev = mock mode**, so `App(language: 'en')` with no args serves mock JSON. No custom deps needed.
- `App` self-wraps the full provider tree (proven by `test/main/app_test.dart` pumping `App` directly).
- Login keys (`lib/core/testing/app_key_constants.dart`): `loginTextFieldUsernameKey`, `loginTextFieldPasswordKey`, `loginButtonSubmitKey`.
- Mock auth: `assets/mock/POST_authenticate.json` → `{ "id_token": "MOCK_TOKEN", ... }` (any credentials succeed).
- Dashboard shell: `ResponsiveScaffold` (`lib/app/shell/responsive_scaffold.dart`).
- `integration_test/` does NOT get `test/flutter_test_config.dart` → the smoke does its own setup via `TestEnv.reset()` (`import '../test/support/test_env.dart';`).
- `flutter test integration_test/` runs headless (no emulator).

## Task 1 (spike): boot → login screen

**Files:**
- Create: `integration_test/app_smoke_test.dart`

- [ ] **Step 1: Write the boot-to-login spike**

```dart
import 'package:flutter/material.dart';
import 'package:flutter_bloc_advance/app/shell/responsive_scaffold.dart';
import 'package:flutter_bloc_advance/core/testing/app_key_constants.dart';
import 'package:flutter_bloc_advance/main/app.dart';
import 'package:flutter_test/flutter_test.dart';
import 'package:integration_test/integration_test.dart';

import '../test/support/test_env.dart';

void main() {
  IntegrationTestWidgetsFlutterBinding.ensureInitialized();

  setUp(() => TestEnv.reset()); // secure-storage mock + storage reset (no global bootstrap here)

  testWidgets('cold start lands on the login screen (mock mode)', (tester) async {
    await tester.pumpWidget(const App(language: 'en'));
    await tester.pumpAndSettle();

    expect(find.byKey(loginTextFieldUsernameKey), findsOneWidget);
    expect(find.byKey(loginButtonSubmitKey), findsOneWidget);
  });
}
```

Run: `fvm flutter test integration_test/app_smoke_test.dart -r compact 2>&1 | tail -5`
Expected: `All tests passed!`.

## Task 2: login → dashboard

**Files:**
- Modify: `integration_test/app_smoke_test.dart`

- [ ] **Step 1: Extend the smoke through login to the dashboard**

Replace the single test with the full flow (keep the boot assertions, then drive login):

```dart
  testWidgets('cold start → login → dashboard (mock mode)', (tester) async {
    await tester.pumpWidget(const App(language: 'en'));
    await tester.pumpAndSettle();

    // Login screen
    expect(find.byKey(loginButtonSubmitKey), findsOneWidget);

    await tester.enterText(find.byKey(loginTextFieldUsernameKey), 'admin');
    await tester.enterText(find.byKey(loginTextFieldPasswordKey), 'admin');
    await tester.tap(find.byKey(loginButtonSubmitKey));
    await tester.pumpAndSettle();

    // Mock auth (POST_authenticate.json → MOCK_TOKEN) → router redirects to the shell
    expect(find.byType(ResponsiveScaffold), findsOneWidget);
  });
```
